Introducing IT Mill Toolkit 5

Effortless web application development for Java-programmers. Visit itmill.com

Release notes version (beta)

Version of IT Mill Toolkit Release 5 is still in beta phase, meaning that the library should be considered unstable and is recommended for evaluation and early adoption. Also, any changes are possible, including API changes, before the stable release.

Summary of changes between IT Mill Toolkit 5.0 and 5.1

See detailed ChangeLog below for a complete list of changes.

General

  • Numerous stability improvements towards a stable Release 5.
  • Support for upcoming IT Mill Testing Tools
  • Manual updated further towards Toolkit 5
  • Upgraded Jetty Servlet Container for demo applications and QuickStart project. Web application is no longer copied to a temporary folder during startup, which should speed up startup and make debugging on Eclipse work better.

Changed Behavior

  • ComboBox component no longer in newItemsAllowed mode by default. Important, may break applications.
  • Adding a component first to A and then to B will move the component from A to B

Notable API Enhancements

  • All components now implement Sizeable interface (Component interface extends Sizeable)
  • GridLayout?'s setWidth/Height now means the functions in Sizeable. Previously they were used to set rows and cols in grid

Deprecated

  • Deprecated setFocusedComponent() and consumeFocus() in Application class, because they had a design flaw and have to be redesigned later. The getFocusedComponent() had very different behaviour than was expected and was removed altogether.
  • Deprecated methods that used a string parameter to do Java reflect. They allowed easy but bad and error prone programming style, which does not survive re-factoring.

Notable Fixes

  • Fixed significant memory leaks and should be clean now.
  • Removed remaining uses of WeakHashMap, allowing better compatibility with Terracotta

See detailed ChangeLog below for a complete list of changes.

Known Issues in (beta)

Installation Issues

  • At least with Windows XP default unzipper or when using WinRAR to uncompress the installation package, uncompression can result in an error such as "The system cannot find the file specified." This is because the uncompressor is unable to handle long file paths where the total length exceeds 256 characters. This occurs, for example, if you try to uncompress the package under Desktop. You should uncompress the package directly under C:\ or some other short path.
  • In Windows, you must have Java Runtime Environment (JRE) installed or the batch file will fail and close immediately.
